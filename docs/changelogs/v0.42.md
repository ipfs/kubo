### 🔦 Highlights

#### Cleaner address announcements for `server` profile nodes

Kubo nodes running the [`server` profile](https://github.com/ipfs/kubo/blob/master/docs/config.md#server-profile) no longer announce loopback or non-globally-reachable IPv6 addresses. This tightens the `server` profile in response to the behavior change announced in [v0.40.0 release notes](https://github.com/ipfs/kubo/blob/master/docs/changelogs/v0.40.md#-libp2p-announces-all-interface-addresses), where libp2p started returning addresses from all network interfaces (including loopback and unallocated IPv6 space such as `1e::/16`, tracked in [go-libp2p#3460](https://github.com/libp2p/go-libp2p/issues/3460)).

Two entries are added to the profile defaults:

- `/ip4/127.0.0.0/ipcidr/8`: IPv4 loopback, per [RFC 1122 §3.2.1.3](https://datatracker.ietf.org/doc/html/rfc1122#section-3.2.1.3).
- `/ip6/::/ipcidr/3`: everything outside global unicast `2000::/3`, per [RFC 4291 §2.4](https://datatracker.ietf.org/doc/html/rfc4291#section-2.4). Covers IPv6 loopback (`::1`), unspecified, IPv4-mapped, NAT64, and unallocated IPv6 space.

> [!NOTE]
> Profiles apply only at init or when re-applied, so existing nodes that set the `server` profile before v0.42 keep the old filter list. Run `ipfs config profile apply server` to pick up the new entries (idempotent, duplicates are skipped), or append the two prefixes manually to [`Swarm.AddrFilters`](https://github.com/ipfs/kubo/blob/master/docs/config.md#swarmaddrfilters) and [`Addresses.NoAnnounce`](https://github.com/ipfs/kubo/blob/master/docs/config.md#addressesnoannounce).
>
> Nodes that peer over IPv6 loopback between co-located daemons, connect to [Yggdrasil](https://yggdrasil-network.github.io/) mesh, or rely on NAT64 should remove `/ip6/::/ipcidr/3` from both fields after applying. See [notes on `/ip6/::/ipcidr/3`](https://github.com/ipfs/kubo/blob/master/docs/config.md#notes-on-ip6ipcidr3) for details.
